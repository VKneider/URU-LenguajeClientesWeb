Introducción a Flexbox

Flexbox (Flexible Box Module) es un modelo de diseño unidimensional en CSS que ofrece una forma más eficiente de disponer, alinear y distribuir el espacio entre los elementos de un contenedor, incluso cuando su tamaño es desconocido o dinámico.

Introducido como parte de CSS3, Flexbox resuelve muchos de los problemas de diseño que eran difíciles o imposibles de resolver con métodos tradicionales como floats o posicionamiento.

¿Por qué usar Flexbox?

  • Simplifica la creación de diseños flexibles y responsivos
  • Permite alinear elementos tanto horizontal como verticalmente
  • Facilita la distribución de espacio y la gestión del tamaño de los elementos
  • Otorga control sobre el orden de aparición de los elementos sin modificar el HTML
  • Soluciona muchos problemas comunes de diseño web

Nota: Flexbox está diseñado para layouts unidimensionales (en una fila o columna), mientras que CSS Grid (que veremos en otra guía) está diseñado para layouts bidimensionales (filas y columnas simultáneamente).

Conceptos Básicos de Flexbox

Para trabajar con Flexbox, es importante entender sus conceptos fundamentales y la terminología asociada.

Terminología clave

/* Crear un contenedor flexible */
.contenedor {
    display: flex;
}

/* Los elementos hijos se convierten 
   automáticamente en flex items */
  • Flex Container (Contenedor Flex): El elemento padre al que se le aplica display: flex.
  • Flex Items (Elementos Flex): Los hijos directos del contenedor flex.
  • Main Axis (Eje Principal): El eje principal del contenedor flex, que va desde el inicio (main-start) hasta el final (main-end).
  • Cross Axis (Eje Transversal): El eje perpendicular al eje principal.
  • Main Size (Tamaño Principal): El ancho o alto del elemento flex en la dirección del eje principal.
  • Cross Size (Tamaño Transversal): El ancho o alto del elemento flex en la dirección del eje transversal.

Ejes y direcciones

La dirección de los ejes en Flexbox depende de la propiedad flex-direction:

/* Eje principal horizontal (predeterminado) */
.contenedor-row {
    display: flex;
    flex-direction: row;
}

/* Eje principal vertical */
.contenedor-column {
    display: flex;
    flex-direction: column;
}

Con flex-direction: row:

  • Eje principal: horizontal (izquierda a derecha)
  • Eje transversal: vertical (arriba a abajo)

Propiedades de los Flex Items

Estas propiedades se aplican a los elementos hijos del contenedor flex (los flex items) y afectan únicamente al elemento al que se aplican.

order

Controla el orden en que aparecen los flex items. Por defecto, todos los elementos tienen order: 0.

/* El valor predeterminado es 0 */
.primero {
    order: 1; /* Aparecerá después */
}

.segundo {
    order: 2; /* Aparecerá al final */
}

.tercero {
    order: -1; /* Aparecerá primero */
}

Orden visual ≠ Orden en el HTML:

1 (order: 1)
2 (order: 2)
3 (order: -1)

flex-grow

Define la capacidad de un flex item para crecer si hay espacio disponible. Acepta un valor numérico sin unidades que sirve como proporción.

/* El valor predeterminado es 0 (no crece) */
.no-grow {
    flex-grow: 0;
}

.grow-1 {
    flex-grow: 1; /* Crece en proporción 1 */
}

.grow-2 {
    flex-grow: 2; /* Crece el doble que grow-1 */
}
flex-grow: 0
flex-grow: 1
flex-grow: 2

flex-shrink

Define la capacidad de un flex item para encogerse si es necesario. Funciona como un factor de contracción.

/* El valor predeterminado es 1 (se encoge) */
.shrink-default {
    flex-shrink: 1;
}

.no-shrink {
    flex-shrink: 0; /* No se encoge */
}

.shrink-more {
    flex-shrink: 2; /* Se encoge el doble */
}

Cuando el contenedor es demasiado pequeño:

shrink: 1
shrink: 0
shrink: 2

flex-basis

Define el tamaño inicial de un flex item antes de que el espacio restante se distribuya según flex-grow y flex-shrink.

/* Puede ser un valor en px, %, em, etc. */
.basis-100 {
    flex-basis: 100px;
}

.basis-50-percent {
    flex-basis: 50%;
}

/* Valor predeterminado */
.basis-auto {
    flex-basis: auto; /* Tamaño basado en el contenido */
}
flex-basis: 100px
flex-basis: 200px
flex-basis: auto

flex

Propiedad abreviada para flex-grow, flex-shrink y flex-basis. Es recomendable usar esta propiedad en lugar de las propiedades individuales.

/* flex: flex-grow flex-shrink flex-basis */

/* Predeterminado */
.flex-default {
    flex: 0 1 auto;
}

/* Igual crecimiento */
.flex-equal {
    flex: 1;  /* Equivale a flex: 1 1 0% */
}

/* Crecimiento proporcional */
.flex-double {
    flex: 2;  /* Equivale a flex: 2 1 0% */
}

/* No crece, no se encoge, tamaño fijo */
.flex-fixed {
    flex: 0 0 200px;
}
flex: 1
flex: 2
flex: 0 0 200px

align-self

Permite sobrescribir la alineación especificada por align-items para flex items individuales.

/* Valores idénticos a align-items */
.align-self-start {
    align-self: flex-start;
}

.align-self-center {
    align-self: center;
}

.align-self-end {
    align-self: flex-end;
}

center
flex-end
stretch (default)

Conclusiones y Buenas Prácticas

Flexbox ha revolucionado la forma en que creamos diseños en CSS, facilitando soluciones a problemas que antes eran complejos de resolver.

Ventajas de Flexbox

  • Simplifica enormemente el centrado vertical, que solía ser muy complicado
  • Facilita la creación de diseños responsivos sin necesidad de media queries complejas
  • Permite distribuir el espacio de forma proporcional o equitativa
  • Ofrece control sobre el orden visual de los elementos sin modificar el HTML
  • Es compatible con todos los navegadores modernos

Buenas prácticas al usar Flexbox

  1. No anides contenedores flex innecesariamente. La simplicidad es clave.
  2. Utiliza la propiedad abreviada flex en lugar de las propiedades individuales cuando sea posible.
  3. Considera la accesibilidad al cambiar el orden visual. El orden del DOM sigue siendo importante para lectores de pantalla.
  4. Usa flex-basis en lugar de width/height para dimensionar flex items.
  5. Combina Flexbox con Grid para diseños más complejos. Flexbox para filas/columnas, Grid para layouts bidimensionales.
  6. Prueba diferentes navegadores y dispositivos para asegurarte de que tu diseño funciona correctamente.

Cuándo usar Flexbox vs. Grid

Usa Flexbox cuando:

  • Necesites alinear elementos en una sola dimensión (fila o columna)
  • El diseño deba adaptarse al contenido
  • Necesites control sobre la alineación
  • Quieras distribuir espacio entre elementos

Usa Grid cuando:

  • Necesites un layout bidimensional (filas y columnas)
  • El diseño deba ser explícitamente definido
  • Tengas un diseño complejo que se beneficie de áreas con nombre
  • Necesites control preciso sobre la posición de elementos

Nota final: Flexbox es una herramienta poderosa que se complementa perfectamente con otras técnicas de CSS como Grid. La combinación de ambas permite crear prácticamente cualquier diseño imaginable de forma eficiente y mantenible.
