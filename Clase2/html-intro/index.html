¿Qué es HTML?

El HTML (HyperText Markup Language) es el lenguaje estándar utilizado para crear y diseñar páginas web. No es un lenguaje de programación, sino un lenguaje de marcado que define la estructura y el contenido de una página web mediante elementos HTML.

Características principales de HTML

  • Estructura: Define la estructura básica de una página web
  • Semántica: Proporciona significado a las partes del contenido
  • Enlaces: Permite conectar diferentes páginas mediante hipervínculos
  • Multimedia: Permite incluir imágenes, audio, video y otros tipos de contenido
  • Formularios: Facilita la interacción del usuario mediante entrada de datos

Evolución de HTML

HTML ha evolucionado significativamente desde su creación:

  • HTML (1991): Creado por Tim Berners-Lee, la primera versión era muy simple
  • HTML 2.0 (1995): Primera especificación estándar de HTML
  • HTML 3.2 (1997): Incorporó tablas y más características
  • HTML 4.01 (1999): Mejoró el soporte para CSS y scripts
  • XHTML (2000): Versión más estricta de HTML usando reglas XML
  • HTML5 (2014): La versión actual, con numerosas mejoras y nuevos elementos semánticos

¿Sabías que...?

Tim Berners-Lee inventó el HTML mientras trabajaba en el CERN (Organización Europea para la Investigación Nuclear) como una forma de compartir documentos entre científicos. Su visión original era crear un sistema de hipertexto que permitiera vincular y acceder a documentos a través de Internet.

¿Cómo funciona HTML?

Cuando visitas una página web, tu navegador descarga el archivo HTML desde el servidor web y lo interpreta para mostrar el contenido formateado en tu pantalla.

Flujo básico:

  1. El navegador solicita un archivo HTML de un servidor.
  2. El servidor envía el archivo HTML al navegador.
  3. El navegador lee el HTML y construye el DOM (Document Object Model).
  4. El navegador muestra el contenido basado en el DOM, aplicando estilos CSS si están disponibles.
  5. El navegador ejecuta cualquier JavaScript para hacer la página interactiva.
<!DOCTYPE html> <html> <head> <title>Mi primera página web</title> </head> <body> <h1>Hola, mundo!</h1> <p>Esta es mi primera página web.</p> </body> </html>

Resultado:

Hola, mundo!

Esta es mi primera página web.

Importante

HTML por sí solo define la estructura y el contenido, pero no el diseño visual o la interactividad. Para eso necesitamos CSS (para el diseño) y JavaScript (para la interactividad).

Elementos HTML

Los elementos HTML son los bloques de construcción de cualquier página web. Un elemento HTML generalmente consiste en una etiqueta de apertura, contenido y una etiqueta de cierre.

Elemento HTML
<p class="texto">
Este es el contenido del párrafo.
</p>

Anatomía de un elemento HTML

  • Etiqueta de apertura: Marca el inicio del elemento y puede incluir atributos (<p class="texto">)
  • Contenido: El texto o los elementos anidados entre las etiquetas
  • Etiqueta de cierre: Marca el final del elemento (</p>)

Elementos vacíos

Algunos elementos no tienen contenido ni etiqueta de cierre. Se conocen como elementos vacíos o de autocierre.

Ejemplos:

  • <img src="imagen.jpg" alt="Descripción"> (Imagen)
  • <br> (Salto de línea)
  • <hr> (Línea horizontal)
  • <input type="text"> (Campo de entrada)

En HTML5, no es necesario cerrar estos elementos, pero en XHTML se cierran con una barra diagonal: <img src="imagen.jpg" alt="Descripción" />

Atributos HTML

Los atributos proporcionan información adicional sobre los elementos HTML y siempre se especifican en la etiqueta de apertura.

<a href="https://www.ejemplo.com" target="_blank">Visita nuestro sitio</a>

En este ejemplo:

  • href es un atributo que especifica la URL del enlace
  • target es un atributo que especifica dónde abrir el enlace

Tipos de atributos

  • Atributos requeridos: Son necesarios para que el elemento funcione correctamente (como src en <img>)
  • Atributos opcionales: Proporcionan funcionalidad o información adicional
  • Atributos globales: Se pueden usar en cualquier elemento HTML (como class, id, style)
  • class: Especifica una o más clases para aplicar estilos CSS
  • id: Proporciona un identificador único para el elemento
  • style: Define estilos CSS en línea
  • title: Proporciona información adicional (aparece como tooltip)
  • lang: Especifica el idioma del contenido
  • data-*: Atributos personalizados para almacenar datos privados
  • aria-*: Atributos para mejorar la accesibilidad

Buenas prácticas

Los valores de los atributos siempre deben estar entre comillas (simples o dobles). Aunque HTML5 permite omitirlas en algunos casos, es una buena práctica incluirlas para mayor claridad y compatibilidad.

Anidamiento de elementos

Los elementos HTML pueden contener otros elementos, creando una estructura jerárquica.

<div class="contenedor"> <h2>Subtítulo</h2> <p>Este es un párrafo con <strong>texto destacado</strong> y un <a href="#">enlace</a>.</p> </div>

Reglas de anidamiento

Los elementos deben anidarse correctamente. Esto significa que un elemento hijo debe cerrarse antes que su elemento padre.

Correcto: <p>Texto con <strong>énfasis</strong></p>

Incorrecto: <p>Texto con <strong>énfasis</p></strong>

Estructura Básica de un Documento HTML

Todo documento HTML debe tener una estructura básica que incluye las siguientes etiquetas fundamentales:

<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Título de la página</title> </head> <body> </body> </html>

Explicación de cada elemento:

  • <!DOCTYPE html>: Declaración del tipo de documento. Indica al navegador que el documento es HTML5.
  • <html>: El elemento raíz que envuelve todo el contenido HTML. El atributo lang especifica el idioma del documento.
  • <head>: Contiene metadatos sobre el documento (información que no se muestra directamente en la página).
  • <body>: Contiene todo el contenido visible de la página web.

El elemento <head>

El elemento <head> contiene metadatos (datos sobre datos) y referencias a recursos externos. La información en el <head> no se muestra en la página, pero es crucial para el funcionamiento correcto.

Elementos comunes dentro de <head>

<head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Título de la página</title> <meta name="description" content="Descripción de la página para motores de búsqueda"> <meta name="keywords" content="HTML, tutorial, web"> <meta name="author" content="Tu Nombre"> <link rel="stylesheet" href="styles.css"> <link rel="preconnect" href="https://fonts.googleapis.com"> <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Roboto&display=swap"> <script src="script.js"></script> <link rel="icon" href="favicon.ico" type="image/x-icon"> </head>

Metaetiquetas importantes

  • <meta charset="UTF-8">: Define la codificación de caracteres del documento.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: Esencial para el diseño responsivo, ajusta la escala en dispositivos móviles.
  • <meta name="description" content="...">: Proporciona una descripción que aparece en los resultados de búsqueda.

¿Por qué es importante el <head>?

Un elemento <head> bien configurado mejora:

  • SEO (optimización para motores de búsqueda)
  • Compatibilidad entre navegadores
  • Rendimiento de la página
  • Experiencia del usuario en dispositivos móviles

El elemento <body>

El elemento <body> contiene todo el contenido visible de la página web, incluyendo texto, imágenes, enlaces, tablas, listas, y más.

<body> <header> <h1>Título Principal</h1> <nav> <ul> <li><a href="#">Inicio</a></li> <li><a href="#">Acerca de</a></li> <li><a href="#">Contacto</a></li> </ul> </nav> </header> <main> <section> <h2>Sección 1</h2> <p>Contenido de la sección 1.</p> </section> <section> <h2>Sección 2</h2> <p>Contenido de la sección 2.</p> </section> </main> <footer> <p>© 2025 Mi Sitio Web</p> </footer> </body>

Estructura semántica

HTML5 introdujo elementos semánticos que dan significado a las diferentes partes de la página:

  • <header>: Encabezado de la página o sección
  • <nav>: Navegación del sitio
  • <main>: Contenido principal
  • <section>: Sección de contenido
  • <article>: Contenido independiente
  • <aside>: Contenido secundario o relacionado
  • <footer>: Pie de página

Recordatorio

Solo puede haber un elemento <body> en un documento HTML. Todo el contenido visible debe estar dentro de este elemento.

¿Qué es el DOM?

El Document Object Model (DOM) es una representación en memoria de un documento HTML como un árbol de nodos. El DOM proporciona una manera de acceder, manipular y modificar el contenido, la estructura y el estilo de un documento HTML.

Características del DOM

  • Es una interfaz de programación para documentos HTML y XML
  • Representa la página para que los programas puedan cambiar la estructura, estilo y contenido
  • Es independiente del lenguaje de programación (aunque comúnmente se accede con JavaScript)
  • Organiza el documento como un árbol de nodos

Estructura del árbol DOM

El DOM representa cada elemento HTML como un nodo en un árbol jerárquico. Esto permite a los desarrolladores navegar fácilmente por la estructura del documento.

Visualización del árbol DOM

Para el siguiente código HTML:

<!DOCTYPE html> <html> <head> <title>Mi página</title> </head> <body> <h1>Bienvenido</h1> <div id="contenido"> <p>Este es un párrafo.</p> <p>Este es otro párrafo.</p> </div> </body> </html>

El árbol DOM se vería así:

Tipos de nodos en el DOM

  • Nodo Documento: La raíz del árbol DOM (document)
  • Nodo Elemento: Representa un elemento HTML (p, div, h1, etc.)
  • Nodo Texto: Contiene el texto dentro de un elemento
  • Nodo Atributo: Representa un atributo de un elemento
  • Nodo Comentario: Representa un comentario HTML

Representación visual

Puedes ver el DOM de cualquier página web usando las herramientas de desarrollo del navegador. En la mayoría de los navegadores, puedes acceder a ellas presionando F12 o haciendo clic derecho en la página y seleccionando "Inspeccionar".

Manipulación del DOM

JavaScript permite manipular el DOM para crear páginas web dinámicas. Algunas operaciones comunes incluyen:

// Seleccionar por ID const elemento = document.getElementById('miId'); // Seleccionar por clase const elementos = document.getElementsByClassName('miClase'); // Seleccionar por etiqueta const parrafos = document.getElementsByTagName('p'); // Selectores CSS (modernos) const elemento2 = document.querySelector('#miId'); // Primer elemento que coincide const elementos2 = document.querySelectorAll('.miClase'); // Todos los elementos que coinciden
// Cambiar el contenido elemento.textContent = 'Nuevo texto'; elemento.innerHTML = 'Texto con HTML'; // Cambiar atributos elemento.setAttribute('class', 'nueva-clase'); elemento.id = 'nuevoId'; // Cambiar estilos elemento.style.color = 'red'; elemento.style.fontSize = '18px';
// Crear un nuevo elemento const nuevoElemento = document.createElement('p'); nuevoElemento.textContent = 'Este es un nuevo párrafo'; // Añadir el elemento al DOM document.body.appendChild(nuevoElemento); // Eliminar un elemento elemento.remove(); // O elemento.parentNode.removeChild(elemento);

Rendimiento del DOM

Manipular el DOM puede ser costoso en términos de rendimiento. Es recomendable minimizar el número de operaciones del DOM agrupando cambios y utilizando fragmentos de documento cuando sea posible.

Importancia del DOM

El DOM es fundamental para:

  • Desarrollo web dinámico: Permite crear páginas interactivas
  • Single Page Applications (SPAs): Aplicaciones que actualizan el contenido sin recargar la página
  • Frameworks modernos: React, Angular y Vue.js trabajan manipulando el DOM (aunque utilizan técnicas de DOM virtual para optimizar)
  • Validación de formularios: Permite validar y modificar datos en tiempo real
  • Animaciones y efectos: Posibilita cambios visuales sin recargar la página

DOM vs. HTML

Es importante distinguir entre el código HTML y el DOM:

  • El HTML es el código fuente que se carga inicialmente
  • El DOM es la representación en memoria después de que el navegador procesa el HTML
  • El DOM puede ser diferente del HTML original debido a la corrección de errores del navegador, JavaScript, o contenido generado dinámicamente

Etiquetas de contenedor

HTML Semántico

Utiliza etiquetas semánticas (<header>, <section>, <article>, etc.) en lugar de <div> genéricos cuando sea posible. Esto mejora la accesibilidad, el SEO y la mantenibilidad del código.

Etiquetas de tabla

Estas etiquetas permiten crear tablas para mostrar datos tabulares.

<table>
Define una tabla.
<tr>
Define una fila de la tabla.
<th>
Define una celda de encabezado de la tabla.
<td>
Define una celda de datos de la tabla.
<thead>
Agrupa el contenido de encabezado de la tabla.
<tbody>
Agrupa el contenido principal de la tabla.
<tfoot>
Agrupa el contenido de pie de la tabla.

Etiquetas de formulario

Estas etiquetas permiten crear formularios interactivos para recopilar datos del usuario.

<form>
Define un formulario.
<form action="/procesar" method="post"> </form>
<input>
Campo de entrada (texto, número, checkbox, etc.).
<input type="text" name="nombre"> <input type="checkbox" name="acepto">
<label>
Etiqueta para un elemento de formulario.
<label for="nombre">Nombre:</label> <input type="text" id="nombre">
<select>, <option>
Lista desplegable con opciones.
<textarea>
Campo de texto multilínea.
<button>
Botón interactivo.

Accesibilidad en formularios

Siempre asocia los elementos <label> con sus campos de formulario correspondientes usando el atributo for y asegúrate de que todos los campos tengan nombres claros. Esto mejora la experiencia para usuarios con tecnologías de asistencia.
